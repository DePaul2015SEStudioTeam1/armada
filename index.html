Armada

Application that monitors docker containers.

Welcome to Armada.

Armada is an open-source project released under the MIT license. It is a Spring/Hibernate based project that monitors and manages your docker containers.

Armada has two main components: core "Armada" and the "Agent(s)".

Armada runs as a deployable Jetty jar, that wraps a spring based service for collecting container log data from different Agents. Your Agents will run locally on whichever server you wish to track containers on.

Armada will consume data from the Agents, storing them in a MySQL database, and provides an operations API for a Javascript front end, currently included as well in the Jetty server.

The Armada front end can be accessed at localhost:8083 and features a dashboard showing active containers across all your environments and real-time statistics that display CPU, memory and disk usage for each container.

How to Deploy

In order to use armada you will have to download the armada.jar and the agent.jar

Armada should be copied to the machine that will act as the server and a central location that will collect data fromt eh agents.

Agents should be deployed on all the machines that are running docker containers

Beside armada.jar and agent.jar you will also need to setup a mysql database. You will need to create a schema called "armada" and a user "armada" with a password "armada". This user should have permissions to access the "armada" schema. Schema setup will occur automatically when you first start armada.

The other tool that you will need to deploy is cAdvisor. cAdvisor is used by the agent to collect metrics from the docker containers.

Armada can be started by simply invoking:

$ java -jar armada.jar &

Agents can be started by invoking:

$ java -jar agent.jar http://armada:8083/
